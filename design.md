# Kadirhan Emre Memiş — Portfolio Design System

> Generated by **anydesign** skill • Brand discipline by **brand-guidelines** • Theme baseline by **theme-factory**.
> Aesthetic: **"Cloud Console — Mission Control"** — dark, technical, Kubernetes-flavoured, but with editorial calm.

## 1. Aesthetic positioning

| Axis | Decision | Why |
|---|---|---|
| Mood | Calm, technical, premium | Cloud/DevOps recruiters skim → trust before flair |
| Tone | Bilingual-safe English, monospace accents | International EU/PL hiring market |
| Density | Generous whitespace, large type | Reads as *senior* not *student* |
| Movement | Lenis smooth scroll + subtle Framer reveals | Modernity without theatrics |
| Reference points | Vercel docs · Cloudflare dashboard · Linear · Istio.io | Industry-native ops feeling |

## 2. Color tokens

All colors expressed as `R G B` triplets so we can drive opacity via Tailwind v4 / `rgb(var(--x) / <alpha>)`.

```
/* Surface */
--rgb-bg:        7 12 22       /* #070C16  near-black with cool blue cast */
--rgb-surface:   13 20 33      /* #0D1421 */
--rgb-surface-2: 19 28 45      /* #131C2D */
--rgb-border:    33 47 71      /* #212F47 */

/* Ink (text) */
--rgb-ink-50:    229 235 245
--rgb-ink-200:   178 191 213
--rgb-ink-400:   123 138 165
--rgb-ink-600:   86 102 132

/* Brand */
--rgb-brand-primary: 56 189 248   /* sky-400  — primary cyan, "cloud" */
--rgb-brand-accent:  16 185 129   /* emerald-500 — "uptime / healthy" */
--rgb-brand-warn:    245 158 11   /* amber-500 — for incident chips */
--rgb-brand-danger:  244 63 94    /* rose-500 — sparingly */
```

Gradient: `linear-gradient(135deg, rgb(var(--rgb-brand-primary)/0.15), rgb(var(--rgb-brand-accent)/0.10))` for hero glow.

## 3. Typography

| Role | Font | Use |
|---|---|---|
| Display | **Poppins** 700 | Hero name, section H2 |
| Body | **Inter** 400/500/600 | Paragraphs, nav, cards |
| Mono | **JetBrains Mono** 500 | Tech badges, code chips, terminal lines |

Scale (rem):
`12 → 14 → 16 → 18 → 20 → 24 → 30 → 36 → 48 → 60 → 80`

Tracking: display `-0.02em`, body `0`, mono `0`.

## 4. Layout grid

- Container: `max-w-6xl` (1152px) with `px-6 md:px-10`.
- Section vertical rhythm: `py-24 md:py-32`.
- Background: full-bleed grid (SVG, 48px cells, 4% opacity) + radial glow behind hero.

## 5. Component inventory

| Component | Pattern |
|---|---|
| `Navbar` | Sticky pill nav, glass blur, logo monogram `KEM/`, nav anchors with mono indices `01·02·03` |

## 6. Motion language

- Section reveal: `opacity 0 → 1`, `y +24 → 0`, `duration 0.6`, `easeOut`, with `viewport={{ once:true, margin:"-15%" }}`.
- Hover on ProjectCard: lift `-4px`, border glow `rgb(var(--rgb-brand-primary)/0.4)`.
- Hero status card: pulsing `▲ uptime 99.99%` dot.
- Lenis: `lerp 0.1`, `wheelMultiplier 1`.

## 7. Accessibility

- Min contrast 4.5:1 for body, 3:1 for large display (verified ink-50 on bg = ~17.4:1).
- Focus visible: 2px outline `rgb(var(--rgb-brand-primary))` offset 2.
- All interactive elements have `aria-label` or visible text.
- Reduced motion respected (Lenis disables, Framer uses `useReducedMotion`).

## 8. Reconstruction notes

Drop-in for the existing `kem-portfolio` Vite scaffold:
1. Wire tokens in `src/index.css` under a `@theme` block (Tailwind v4) + `:root`.
2. Build sections in this order: Hero → About → TechStack → Projects → Experience → Certifications → Contact.
3. Drive content from `src/data/*.ts` so future edits don't touch JSX.
4. Use `framer-motion` lazy `m` API only after profiling; stock `motion` is fine for this scale.
